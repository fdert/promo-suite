# واجهة نظام إدارة الوكالة (مبنية من الصفر)

مشروع React + Vite + Tailwind جديد بالكامل، يحل محل الواجهة القديمة (التي كانت مبنية مسبقًا ومصغّرة/غير قابلة للتعديل). يتواصل مع نفس الـ Backend (`api/index.php`) من المرحلتين السابقتين (الأمان + عزل الوكالات).

## ⚠️ ملاحظة مهمة جدًا قبل البدء
تم كتابة هذا الكود بالكامل يدويًا **بدون القدرة على تشغيل `npm install` أو تجربة البناء فعليًا**، لأن بيئة العمل هنا لا تملك اتصال إنترنت لتحميل الحزم. راجعت كل ملف يدويًا وتأكدت من تطابق الأقواس وصحة الاستيرادات، لكن **لم يُختبر التشغيل الفعلي بعد**. أول شيء تفعله:

```bash
npm install
npm run dev
```

ثم افتح المتصفح وراقب الـ console لأي أخطاء. إذا واجهت أي خطأ في البناء، أرسل لي رسالة الخطأ وسأصلحه فورًا.

**الأفضل لمتابعة تطوير هذا المشروع تحديدًا: استخدم Claude Code** (بدلاً من هذه المحادثة) — لأنه يستطيع فعليًا تشغيل `npm install`/`npm run dev` ورؤية الأخطاء وإصلاحها مباشرة، بخلاف بيئة المحادثة هذه.

## البنية
```
src/
  lib/api.js          # عميل API يغلّف نداءات api/index.php (db/auth/functions/storage)
  context/             # AuthContext (تسجيل الدخول)، ToastContext (إشعارات)
  components/          # Layout (الشريط الجانبي RTL)، ProtectedRoute، مكوّنات UI أساسية
  pages/                # Login, Signup, Dashboard, Customers, Orders, Payments, Expenses, Settings
```

## الوحدات المُنجزة حتى الآن
تسجيل الدخول/التسجيل (ينشئ وكالة جديدة تلقائيًا)، لوحة المعلومات، العملاء (CRUD كامل)، الطلبات (CRUD)، المدفوعات، المصروفات، الموظفون والرواتب (CRUD + صرف راتب)، صندوق محادثات واتساب (قائمة + محادثة + إرسال)، خطط التقسيط (بحث عن طلب + إنشاء خطة أقساط)، إعدادات ربط واتساب (WaSender) والذكاء الاصطناعي لكل وكالة، التقارير (التقرير المالي اليومي + تقرير الرواتب)، تقييمات العملاء.

**الجولة الحالية أضافت:**
- **صفحة الأسعار العامة** (`/pricing`) — تعرض الباقات الثلاث لأي زائر قبل التسجيل
- **الدفع عبر Moyasar** (`/billing/checkout` + `/billing/checkout/complete`) — نموذج بطاقة مُستضاف من Moyasar، مع تحقق من حالة الدفع من جهة الخادم فقط (لا يُعتمد على استجابة المتصفح أبدًا) + ويب هوك احتياطي
- **بطاقة الاشتراك والفوترة** في الإعدادات — تعرض الباقة والحالة وتاريخ انتهاء التجربة، مع زر ترقية
- **لوحة إدارة المنصة (Super Admin)** (`/admin`، تظهر فقط لـ `platform_admin`) — نظرة عامة (عدد الوكالات، MRR تقديري)، إدارة الوكالات (تفعيل/إيقاف، تغيير الباقة)، إدارة الباقات (CRUD كامل)

## ⚠️ قبل تفعيل الدفع الحقيقي
1. أنشئ حساب Moyasar وأدخل `MOYASAR_PUBLISHABLE_KEY` و`MOYASAR_SECRET_KEY` و`MOYASAR_WEBHOOK_SECRET` في `api/config.php` (انظر `api/config.example.php` المحدّث في حزمة الـ Backend).
2. اضبط رابط الويب هوك في لوحة Moyasar إلى: `https://yourdomain.com/fdert/api/index.php?service=functions&action=billing-webhook&secret=<MOYASAR_WEBHOOK_SECRET>`
3. **اختبر التدفق كاملاً في وضع Moyasar التجريبي (Test Mode) قبل التفعيل الحقيقي** — لم أستطع اختبار هذا التكامل ضد حساب Moyasar فعلي في هذه الجلسة.

## بهذا اكتمل المشروع بالكامل من ناحية الميزات
كل المراحل الست المتفق عليها أصلًا منجزة: الأمان، عزل الوكالات، الواجهة الأمامية الكاملة، الاشتراك والدفع، لوحة Super Admin. المتبقي فعليًا هو مرحلة عملية بحتة: **بناء أداة النشر التلقائي على Hostinger VPS**.

## الربط بالخادم الخلفي
- في التطوير: `vite.config.js` يوجّه `/api` عبر proxy لخادم PHP محلي أو تجريبي (عدّل `target`).
- في الإنتاج: بعد `npm run build`، ضع محتويات `dist/` في نفس مجلد نشر الـ Backend (`api/`, `.htaccess`, إلخ من الحزمة السابقة)، بحيث يكون `index.html` والأصول بجانب `api/`.

## القيم الافتراضية للتصميم
الخط: Cairo (العناوين) + Tajawal (النصوص) — نفس خطوط النسخة الأصلية. الألوان: خلفية ورقية دافئة (`#F7F6F3`)، شريط جانبي كحلي غامق (`#15161C`)، لون تمييز برتقالي حيوي (`#E8542C`) يعكس طابع وكالة إبداعية دون أن يكون التصميم النمطي المعتاد لواجهات الذكاء الاصطناعي.
